# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

AutoPsychDx is a Python CLI tool that spawns a Claude Code agent inside a tmux session to perform automated psychometric diagnosis. Given item-level response data from any psychological scale, it applies three diagnostic methods (sum-score cut-off, IRT, DCM) and generates a structured report.

## Build & Install

```bash
pipx install -e .          # recommended for CLI use
pip install -e .           # development install
```

R packages (required at runtime, not at install):
```r
install.packages(c("mirt", "CDM"))
```

System dependencies: `tmux`, `R >= 4.0`, Claude Code CLI (`claude`).

## CLI Commands

```bash
diagnosis compile <folder>              # generate items.csv from responses.csv
diagnosis clean <folder>                # remove Output/ (generated results)
diagnosis clean <folder> --all          # also remove items.csv (responses.csv is never deleted)
diagnosis compile <folder> --no-attach  # compile in background with spinner
diagnosis run <folder>                  # run diagnosis pipeline (background spinner, blocks until done)
diagnosis run <folder> --clear          # delete Output/ first, then run
diagnosis attach <name>                 # reattach to running session
diagnosis kill <name>                   # stop a session
diagnosis ls                        # list active sessions
diagnosis version                   # show version
```

## Testing

No unit test suite. CI runs integration tests:

```bash
# Verify CLI installs and runs
pip install -e . && diagnosis --help && diagnosis version

# Test folder validation
diagnosis run Projects/PTSD_Forbes2018 2>&1 || true   # should pass validation
diagnosis run /tmp/nonexistent 2>&1 || true            # should fail with "folder not found"

# Test R data preparation
cd Projects/PTSD_Forbes2018 && Rscript prepare_responses.R
```

CI workflow: `.github/workflows/ci.yml` (two jobs: `test-cli` on Ubuntu/macOS × Python 3.10/3.12, `test-r-script` on Ubuntu with R 4.3).

## Architecture

The pipeline flow is: **CLI validates → builds prompt from skill files → spawns tmux session → Claude agent executes workflow → R scripts run diagnostics → report generated**.

### Key components

- **`diagnosis/cli.py`** — Typer CLI app. `run()` validates the project folder, builds the prompt via `skill_loader`, and calls `tmux.spawn()`.
- **`diagnosis/tmux.py`** — Creates a detached tmux session and runs `claude --dangerously-skip-permissions -p "$(cat prompt_file)"`. Handles attach/detach/kill/poll.
- **`diagnosis/skill_loader.py`** — Loads markdown skill files from `diagnosis/skills/` using `importlib.resources`. `build_prompt()` concatenates them into the agent's system prompt.
- **`diagnosis/skills/diagnosis.md`** — Agent workflow: validate inputs → generate R script → execute → write report. This is the main behavior definition.
- **`diagnosis/skills/psychometric-diagnosis.md`** — R function definitions for all three methods (cut-off, IRT via `mirt`, DCM via `CDM::gdm`) plus the integration pipeline function `run_diagnosis_pipeline()`.

### Skill-file driven design

All agent behavior is defined in markdown skill files under `diagnosis/skills/`. Editing these files changes agent behavior for all projects without touching Python code. Mirror copies exist in `.claude/commands/` for use as Claude Code slash commands.

### Project folder convention

Each analysis project lives under `Projects/<name>/` with:
- `items.csv` (required) — item metadata: `item_id, item_text, scale, cutoff, response_min, response_max`
- `raw_data.csv` (required if `prepare_responses.R` references it) — the original dataset before item selection. The agent needs this file present so `prepare_responses.R` can execute successfully.
- `prepare_responses.R` (optional) — loads raw data, selects item columns, writes `responses.csv`. If missing, the agent generates a template.
- `responses.csv` (generated) — rows = persons, columns = item_ids
- `Output/` (generated) — R scripts, results CSVs, report markdown

`Output/` and `responses.csv` are git-ignored and regenerated by running the pipeline. Raw data files may be downloaded at runtime by `prepare_responses.R` (e.g., from OSF) or must be placed in the project folder manually.

## Important Patterns

- Prompt is written to a temp file before being passed to `claude` to avoid shell quoting issues.
- Session names are derived from folder names: `diagnosis-{folder-name}` (spaces/underscores → hyphens, lowercased).
- The agent writes R scripts that inline all function definitions from the psychometric-diagnosis skill, making each script self-contained.
- Consensus diagnosis: a person is flagged if ≥2 of 3 methods agree.
- Currently unidimensional only (single latent construct). Multidimensional support is on the roadmap.
