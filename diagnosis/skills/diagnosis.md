# Diagnosis Command

You are a psychometric diagnosis agent. Follow the steps below exactly, in order.

---

## Expected Folder Structure

```
project_folder/
  items.csv             # required: item metadata
  responses.csv         # required: persons × items response matrix (can be provided directly)
  prepare_responses.R   # optional: script to generate responses.csv from raw data
  Output/               # auto-created: all output files go here
```

`items.csv` column definitions:
- `item_id`: unique item identifier (e.g., PHQ1, PCL1); must match column names in `responses.csv`
- `item_text`: the full text of the item as presented to respondents
- `scale`: the scale/instrument this item belongs to (e.g., PHQ-9, PCL-5); used to group items and name outputs
- `cutoff`: the validated sum-score cut-off for this scale (one value per scale; repeat across rows)
- `response_min`: minimum possible response value (e.g., 0)
- `response_max`: maximum possible response value (e.g., 3)

---

## Agent Workflow

### Step 0 — Validate items.csv

1. Check that `items.csv` exists in the project folder. If it does not:
   - Ask the user: "items.csv not found. Would you like me to generate it from your raw response data? (yes/no)"
   - If **yes**: follow the `/generate-items` workflow, then continue to Step 1 once `items.csv` has been created.
   - If **no**: report the error and stop. Remind the user that `items.csv` is required and can be generated with `/generate-items`.
2. Read `items.csv` and verify it contains all required columns: `item_id`, `item_text`, `scale`, `cutoff`, `response_min`, `response_max`. Report any missing columns and stop if any are absent.

### Step 1 — Ensure responses.csv exists

If `responses.csv` already exists in the project folder, proceed directly to Step 2.

If `responses.csv` does **not** exist:

1. Check whether `prepare_responses.R` exists. If it does not, generate it. The script must:
   - Download or load the raw data from whatever source is inferable from the project context (README, existing files, or prior conversation).
   - Select the columns corresponding to the `item_id` values listed in `items.csv`.
   - Write the result as `responses.csv` with column names matching `item_id`.

   Template:

   ```r
   # prepare_responses.R — auto-generated by diagnosis agent
   # Edit the data source section below if needed, then re-run diagnosis.

   # --- Data source ---
   # {DESCRIPTION OF DATA SOURCE}

   {DATA_LOADING_CODE}

   # --- Select items listed in items.csv ---
   items    <- read.csv("{PROJECT_FOLDER}/items.csv")
   item_ids <- items$item_id

   responses <- raw_data[, item_ids]

   # --- Save ---
   write.csv(responses, "{PROJECT_FOLDER}/responses.csv", row.names = FALSE)
   message("Saved responses.csv: ", nrow(responses), " persons x ", ncol(responses), " items")
   ```

2. Run `prepare_responses.R`:

   ```bash
   Rscript {PROJECT_FOLDER}/prepare_responses.R
   ```

   If the script errors, report clearly and stop.

### Step 2 — Validate Inputs

1. Read `items.csv` and `responses.csv`.
2. Verify all `item_id` values in `items.csv` appear as column names in `responses.csv`. Report missing ones and stop if any.
3. Extract distinct scales. For each scale record: `scale_name`, `cutoff`, `response_min`, `response_max`, item list.
4. Infer `itemtype` for mirt: `response_max - response_min == 1` → `"2PL"`, otherwise → `"graded"`.
5. Create `{PROJECT_FOLDER}/Output/` if it does not exist.

### Step 3 — Generate Diagnosis R Script

For each scale, generate `{PROJECT_FOLDER}/Output/{scale_name}_diagnosis.R`. The script must:

1. Inline all skill functions from the psychometric-diagnosis skill (`diagnose_cutoff`, `fit_irt`, `fit_dcm`, `compare_methods`, `generate_report`, `run_diagnosis_pipeline`).
2. Load `responses.csv` and subset columns for this scale.
3. Call `run_diagnosis_pipeline()` with arguments derived from `items.csv`.
4. Save person-level results to `{PROJECT_FOLDER}/Output/{scale_name}_diagnosis_results.csv`.
5. Capture printed output with `capture.output()` and save to `{PROJECT_FOLDER}/Output/{scale_name}_diagnosis_output.txt`.

Template:

```r
# Auto-generated by diagnosis agent on {DATE}
# Scale: {SCALE_NAME} | Construct: {CONSTRUCT} | Cut-off: {CUTOFF}

library(mirt)
library(CDM)

# --- Skill functions ---
{INLINE_SKILL_FUNCTIONS}

# --- Load data ---
responses  <- read.csv("{PROJECT_FOLDER}/responses.csv")
scale_data <- responses[, c({ITEM_ID_LIST})]

# --- Output directory ---
dir.create("{PROJECT_FOLDER}/Output", showWarnings = FALSE)

# --- Run pipeline ---
report_output <- capture.output(
  results <- run_diagnosis_pipeline(
    data       = scale_data,
    construct  = "{CONSTRUCT}",
    instrument = "{SCALE_NAME}",
    cutoff     = {CUTOFF},
    itemtype   = "{ITEMTYPE}",
    output_csv = "{PROJECT_FOLDER}/Output/{SCALE_NAME}_diagnosis_results.csv"
  )
)

# --- Save text output ---
writeLines(report_output, "{PROJECT_FOLDER}/Output/{SCALE_NAME}_diagnosis_output.txt")
cat(paste(report_output, collapse = "\n"))
```

### Step 4 — Execute R Script

```bash
Rscript {PROJECT_FOLDER}/Output/{scale_name}_diagnosis.R
```

Capture stdout and stderr. If the script errors, report clearly and stop.

## Notes

- Infer `construct` from scale name: "PHQ-9" → "Depression", "GAD-7" → "Anxiety", "PCL-5" → "PTSD". If uncertain, use the scale name.
- If `cutoff` is missing, skip Method A and note it in the report.
- All output files go inside `{PROJECT_FOLDER}/Output/`. Never write outside it.
- `prepare_responses.R`, `items.csv`, and `responses.csv` stay in the project root (not in Output/).
- `prepare_responses.R` is only needed if `responses.csv` is not already present.
